# Application Statuses

Use a small status set so the pipeline stays readable. The goal is not perfect categorization; the goal is knowing what needs attention next.

| Status | Meaning | Typical next action |
|---|---|---|
| Lead | Interesting company or role found, but not evaluated yet. | Decide whether it is worth researching. |
| Researching | Company, role, hiring manager, or fit is being investigated. | Create or update a company note. |
| Preparing | Materials are being tailored before applying or reaching out. | Draft application, referral ask, or outreach. |
| Applied | Application has been submitted. | Track confirmation and plan follow-up. |
| Recruiter screen | Recruiter or talent conversation is scheduled or completed. | Prepare role-fit story and questions. |
| Interviewing | Product, design, hiring manager, or panel interviews are active. | Prepare stories, examples, and follow-ups. |
| Assignment | A task, case, critique, or presentation is active. | Track deadline, scope, and submission state. |
| Final stage | Late-stage process, references, executive round, or offer discussion. | Prepare decision criteria and negotiation notes. |
| Offer | Offer received or being negotiated. | Compare against criteria and prepare response. |
| Rejected | Company declined or process ended externally. | Capture useful learning, then archive. |
| Withdrawn | Eddie chose to stop pursuing the opportunity. | Record reason, then archive. |
| Archived | No longer active, stale, duplicate, or not relevant now. | Keep only the useful context. |

## Field Guidance

| Field | Guidance |
|---|---|
| Company | Use the company name consistently across tracker, notes, and materials. |
| Role | Use the posted title. Add level notes in the company note if needed. |
| Priority | Use `P0`, `P1`, `P2`, `P3`, or `Unknown`. `P0` means highest active priority; `P1` means strong target; `P2` means possible target or useful signal; `P3` means lower priority or exploratory. Use a range such as `P1/P2` only when imported context is explicitly uncertain. |
| Last update | Date of the last real event or meaningful decision. Use `Around` or `Before` when imported context is approximate. |
| Next action | A concrete verb-led action, not a vague reminder. |
| Due | Date for the next action when timing matters. |
| Links / Notes | Add job post, company note, contact, or material links. |

## Adding A Status

Add a new status only when repeated real cases do not fit the existing model. If a status is just a temporary mood or uncertainty, capture it in notes instead.
