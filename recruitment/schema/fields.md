# Recruitment Data Fields

Use these fields when creating company notes, role evaluations, and application records. Consistent fields make it easier for Codex to help later without re-parsing scattered chat history.

## Privacy Boundary

Use role/contact descriptors in versioned files. Keep exact names, direct contact details, private recruiter notes, interview readouts, compensation numbers, floors, walk-away points, and negotiation strategy in `recruitment/private/`.

## Company

- Name
- Website
- Industry / category
- Location / remote model
- Product surface
- Business model
- Recent signals
- Why this company might fit
- Risks or concerns
- Source links

## Role

- Role title
- Level
- Team / function
- Hiring manager or recruiter descriptor
- Job post link
- Compensation category or public range source
- Required strengths
- Evidence Eddie can use
- Gaps to address
- Application deadline, if known

## Application

- Status
- Priority
- Last update
- Next action
- Due date
- Materials needed
- Contact descriptors
- Safe follow-up history
- Outcome

## Material

- Material type
- Company / role
- Audience
- Core message
- Evidence to include
- Claims to avoid
- Draft location
- Review status
